🛠️ Sticky Note 프로젝트 – 앞으로 해야 할 일

○ DB 구조 확정 및 초기화

- User, MemoBoard, Memo 테이블 생성 ✅

- 기존 테이블이 있으면 DROP 후 재생성 ✅

- 테스트용 데이터(User id=1 기준, 메모보드 2개, 각 보드에 여러 개 메모) 삽입 ✅

○ 불러오기 기능 구현

- 로그인 후 현재 계정(User) 관련 MemoBoard + Memo 데이터를 불러오기 ✅

- 각 Memo의 위치, 크기, 폰트, 색상 등 스타일 정보 포함 ✅

- 불러온 데이터를 프론트엔드에서 렌더링 ✅

○ 저장 기능 구현

- 사용자가 메모 내용, 위치, 크기, 색상, 폰트 등을 변경하면 DB에 업데이트

- Memo 단위 또는 MemoBoard 단위로 저장 가능하도록 구현

○ 메모/메모보드 생성 기능 구현

- 새 MemoBoard 생성

- 새 Memo 생성 및 해당 MemoBoard에 연결

- 생성 즉시 렌더링

○ 삭제 기능 구현

- Memo 삭제 시 DB에서 제거

- MemoBoard 삭제 시 연결된 Memo도 삭제(FK 관계)

○ 메모 이동 및 크기 조절 기능 구현

- 기본적으로 마우스 이동은 막기

- 설정에서 “움직이기” 활성화 시 방향키 + 기능키로 이동 가능

- 크기 조절도 설정에서 활성화/비활성화 가능

○ 스타일 설정 기능 구현

- 글자 크기, 글자 색상, 폰트, 글자 두께, 배경색, 테두리 크기/색상, overflow 옵션

- 각 Memo마다 독립적으로 적용 가능

○ 전체 선택 기능 구현

- Ctrl + A로 선택 시 여러 Memo 중 선택된 것만 전체 선택

- 이후 복사, 삭제 등의 액션 적용 가능

○ 다크모드 적용 범위 결정

- 헤더와 사이트 주변 영역만 다크모드 적용

- MemoBoard, Memo 색상은 사용자 지정 값 유지

○ 프론트엔드와 DB 연결 테스트

- Prisma Client 기준 데이터 불러오기, 생성, 수정, 삭제

- React/Tailwind UI에서 실제 렌더링 확인

○ 추가 기능 / 고도화

- 여러 사용자 계정 동시 사용 테스트

- 메모 검색, 정렬, 필터 기능 추가

- 메모 이동/크기 조절 UX 개선
